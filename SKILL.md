# 小红书内容策略技能

用于解决“做什么号、给谁看、发什么、怎么持续变好”这几个问题。
这个技能偏策略和内容系统，不负责图片渲染或发布；如果任务已经进入成稿、卡片生成或发布阶段，转用 `xhs-note-creator`。

## 什么时候用

- 用户要从零开始做小红书账号。
- 用户已经在发内容，但赛道、定位、内容方向混乱。
- 用户需要一套稳定的内容生产方法，而不是零散技巧。
- 用户问为什么没流量、为什么不涨粉、为什么内容不转化。
- 用户要结合自身经历、产品或服务，设计小红书增长和变现路径。

## 工作方式

### 1. 先判断账号类型

先区分两类账号，再决定后续策略：

- 个人 IP 类：核心是人和观点，常见变现是品牌合作、课程、咨询、买手等。
- 商业类：核心是产品或服务，重点不是大流量，而是精准触达和转化。

如果用户没有说明，优先根据其目标推断：

- 想做个人影响力、副业、接广告，按个人 IP 类处理。
- 想卖货、卖服务、引流成交，按商业类处理。

### 2. 用 3+1 赛道法做定位

定位必须同时看三件事：

- 喜欢：是否愿意长期讲这个主题。
- 擅长：是否比普通人更有经验、方法、故事或审美。
- 需求：小红书上是否已有持续更新且跑得动的账号与内容。

默认输出：

- 3 个主赛道
- 1 个辅赛道
- 目标用户
- 用户痛点
- 账号提供的价值
- 账号的一句话定位

不要鼓励“拍脑袋赛道”或纯自嗨方向。

### 3. 找对标账号，不抄人格

将对标账号分成两类：

- 主对标：赛道和变现方式接近，用来判断方向、天花板、转化模式。
- 次对标：赛道不同，但表达、镜头、封面、结构很强，用来借创意。

输出对标分析时，优先总结：

- 他们在讲什么问题
- 他们主要用什么内容形式
- 哪些主题反复有效
- 哪些表达适合借鉴
- 哪些只是对方人格红利，不适合照搬

### 4. 先搭内容系统，再写单篇内容

默认采用这套框架：

- 爆款笔记 5 要素：选题、文案、拍摄、标题、封面
- 笔记 2 大类型：干货类、情绪类

先做内容系统，再做单篇选题：

- 明确 3-5 个内容支柱
- 为每个支柱拆 10-20 个选题
- 建立选题库和素材库
- 再挑当前最适合的题做成具体内容

写选题时，优先写成这个格式：

`目标用户 + 具体场景/问题 + 承诺结果 + 角度`

例如：

- 刚毕业女生，第一份工作总被 push，怎么快速建立边界感
- 独居打工人下班后极度疲惫，如何用 20 分钟恢复精神

### 5. 内容判断标准

默认按下面顺序判断内容是否成立：

1. 这条内容是在解决问题、提供利益，还是只是在表达自己？
2. 用户是否能在封面和标题里立刻看懂“和我有什么关系”？
3. 开头是否足够快地进入主题？
4. 主体是否有信息密度、故事性或情绪张力？
5. 看完后，用户为什么要赞藏评转或关注？

如果用户只说“帮我做内容”，先给这些产物之一：

- 账号定位卡
- 30 个选题
- 7 天内容排期
- 单篇笔记 brief

如果用户明确要成稿，再转给 `xhs-note-creator` 或直接按平台风格写稿。

### 6. 复盘时先拆指标，不先怪限流

复盘默认从这四层开始：

- 点击率：重点看标题和封面
- 读完率/完播率：重点看开头、结构、节奏
- 互动率：重点看内容是否有价值、是否有观点、是否引发共鸣
- 转粉/转化：重点看账号定位是否清晰、内容是否持续指向同类人群

如果数据差，不要先给“被限流了”结论，先写出更可能的内容原因。

### 7. 变现建议必须和账号阶段匹配

常见变现路径：

- 个人 IP 类：品牌合作、买手、课程、咨询、社群、直播
- 商业类：店铺、私域引流、咨询预约、服务成交、直播带货、商品卡

判断变现方案时同时看：

- 用户意图是否明确
- 客单价高低
- 是否需要信任建立
- 当前粉丝量和内容成熟度

不要默认把“涨粉最多”当成“最赚钱”。

## 默认交付物

根据任务，从下面几种里选最合适的交付：

- 账号定位卡：账号类型、3+1 赛道、目标用户、价值主张、简介草案
- 对标拆解：主对标/次对标、可借鉴点、不可照搬点
- 内容规划：内容支柱、30 个选题、7 天或 14 天排期
- 单篇 brief：选题、标题方向、封面方向、结构、行动号召
- 复盘报告：问题定位、原因判断、下一轮调整动作
- 变现建议：适合当前阶段的变现路径和内容打法
